    Title: A Pilot Study on Secure Code Generation with ChatGPT for Web Applications
    Type: Self-defined tasks

    Approach:
    - This paper presents a pilot study that uses ChatGPT for generating web application code with a specific emphasis on mitigating four prevalent web application vulnerability types.
    - Creation of a simple webapp with 7 microservices that is designed for a university library that provides online access to students, staff, and library personnel.

[47]  Khoury et. al.

    Title: How Secure is Code Generated by ChatGPT?
    Type: Self-defined tasks

    Approach:
    - We asked ChatGPT to generate 21 programs, in 5 different programming languages.
    - We then evaluated the generated program and questioned ChatGPT about any vulnerability present in the code.
    - The programs generated serve a diversity of purpose, and each program was chosen to highlight risks of a specific vulnerability.

    Results:
    - Results suggest that ChatGPT is aware of potential vulnerabilities, but nonetheless often generates source code that are not robust to certain attacks.
    - In fact, only 5 of the 21 use-cases we investigated were initially secure, and ChatGPT was only able to produce secure code in an additional 7 cases after we explicitly requested of it that we correct the code.
    - ChatGPT seems aware of — and indeed readily admits, the presence of critical vulnerabilities in the code it suggests.
    - The code may well contain other vulnerabilities, and indeed, several programs were deemed ’corrected’ even though they contained obvious vulnerabilities, because ChatGPT seems to have corrected the issue we sought to explore in this use-case.

[8]  Asare et. al.

    Title: Is GitHub’s Copilot as Bad as Humans at Introducing Vulnerabilities in Code ?
    Type: Dataset

    Approach:
    - The aim of this study is to determine if Copilot is as bad as human developers.
    - Using a dataset of C/C++ vulnerabilities, we prompt Copilot to generate suggestions in scenarios that led to the introduction of vulnerabilities by human developers.
    - The dataset consists of a total of 3754 C and C++ vulnerabilities across 348 projects.
    - In this study, due to the significant manual effort required to interact with Copilot and analyze its results, we capped our sample size at 153.
    - For each of the selected samples, we curated prompts representing the state of the project before the vulnerability was introduced by the human developer.

    Results:
    - Overall, we conclude that Copilot is not as bad as human developers at introducing vulnerabilities in code.
    - We find that Copilot replicates the original vulnerable code about 33% of the time while replicating the fixed code at a 25% rate.
    - We found that there were some vulnerability types where Copilot was more likely to generate vulnerable code than fixed code.
    - However this behaviour is not consistent: Copilot is more likely to introduce some types of vulnerabilities than others and is also more likely to generate vulnerable code in response to prompts that correspond to older vulnerabilities.
